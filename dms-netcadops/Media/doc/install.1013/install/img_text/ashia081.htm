The figure shows a load balancer fronting two nodes (node 1 and node 2). Each of the nodes run Oracle HTTP Server. Nodes 3 and 4 each run OC4J. The Oracle HTTP Server on node 1 can direct requests to OC4J on node 3 or node 4. Similarly, the Oracle HTTP Server from node 2 can also direct requests to OC4J on node 3 or node 4.

As a variation, you can use Oracle Identity Management from Release 2 (10.1.2) with this topology. You can also use Oracle HTTP Server from Release 2 (10.1.2) with this topology.
